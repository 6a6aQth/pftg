🔶 M — MODEL THE SYSTEM (Requirements & Constraints)
________________________________________
1. Functional Requirements
Core Platform
•	User authentication & account management
•	Farm mapping using GPS coordinates
•	Plot segmentation within farms
•	Crop and input (fertilizer, soil, e.tc ) assignment
________________________________________
Environmental Intelligence
•	Real-time weather data per farm location:
o	Rainfall (amount + timing)
o	Temperature
o	Humidity
o	Solar radiation
________________________________________
Geospatial & NDVI Intelligence (NEW)
The system must:
•	Fetch satellite imagery data
•	Compute and store NDVI values
•	Generate NDVI time-series per farm
•	Detect vegetation health and stress
•	Identify NDVI anomalies (deviation from historical baseline)
•	Track crop phenological stages (growth cycles)
________________________________________
Digital Twin Features
•	Maintain a dynamic digital representation of each farm
•	Continuously update using:
o	Weather data
o	NDVI data
o	User inputs
•	Enable future:
o	Scenario simulation
o	Yield prediction
________________________________________
AI & Insights
•	Generate recommendations based on:
o	Crop type
o	Weather
o	Soil estimation
o	NDVI trends
•	Detect:
o	Crop stress
o	Drought risk
o	Nutrient deficiencies (inferred)
________________________________________
Notifications
•	SMS alerts for:
o	Weather risks
o	Crop stress
o	Action recommendations
________________________________________
2. Non-Functional Requirements
•	Availability: 99.9% uptime
•	Latency: <300ms API response
•	Scalability: Up to 1M+ users
•	Security: Encrypted data + secure auth
•	Observability: Logs, metrics, tracing
•	Cost-efficiency: Optimized for African markets
________________________________________
3. Constraints
•	Limited connectivity (offline-first needed)
•	No real-time IoT sensors initially
•	High cost of satellite raw processing
•	SMS cost constraints
________________________________________
4. Success Metrics
•	NDVI processing latency < 5s (API-based)
•	SMS delivery success >95%
•	User retention
•	Yield improvement indicators

🔷 T — TRADEOFFS
________________________________________
Satellite Data Tradeoffs
Sentinel-2 (10m)
✔ High precision
❌ Heavy processing
________________________________________
Landsat (30m)
✔ Historical depth
❌ Lower resolution
________________________________________
MODIS (250m+)
✔ Fast & lightweight
❌ Low precision
________________________________________
👉 Strategy:
•	Use Sentinel-2 for farms
•	Use MODIS/Landsat for trends
________________________________________
NDVI Processing
Raw processing
❌ Expensive
API-based (GEE / Sentinel Hub)
✔ Efficient
✔ Scalable
👉 Decision: API-first approach
